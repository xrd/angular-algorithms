Steps

  1. First step: heap is built out of the data
  2. Heap is binary tree, but using a simple array.
  3. Algorithm for finding node: 
     
      iParent = floor(i-1)/2
      iLeftChild = 2*i + 1
      iRightChild = 2*i + 2
       
  4. Items are placed into heap, tree structure where top item at a node dominates all others.
  5. After put into heap, items are extracted one by one to build sorted array.

Notes on heaps

  1. All nodes dominate their children
  2. Insertion: put node at the bottom, and then bubble the node up.
  3. Deletion: remove the node, then bubble down the node until it satisfies the dominating properties.

Comparisons

  1. Competes with Quicksort.
  2. Heapsort is better against attacks because it cannot ever be O(n^2) like Quicksort
  3. Embedded systems often use Heapsort because auxiliary storage is bounded and runtime is constrained.
  4. Competes with Merge sort. Merge sort uses
